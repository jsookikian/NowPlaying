typedef void(GNSDK_CALLBACK_API * gnsdk_memory_warn_fn)(gnsdk_void_t *callback_data, gnsdk_size_t cur_mem_size, gnsdk_size_t memory_warn_size)

Sets a callback to notify the application when GNSDK reaches a certain level of memory usage. This callback is informational only. If the application gets this callback, it must decide the right course of action (for example, stop pending queries, or do nothing).

Remarks

The memory warn callback is called the first time GNSDK heap usage crosses memory_warn_size. Afterwards, it will be called for every 1024 byte increase (or more) above the cur_mem_size parameter of the previous call. If memory usage starts to fall, but still remains above the memory_warn_size, the callback will not be called until it reaches a level of 1024 bytes above the previous call. When memory usage falls below the memory_warn_size, the callback is called to indicate usage is now below the set threshold.

Parameters
mem_warn_fn[in] The function to call when memory reaches the level designated by memory_warn_size
p_arg[in] Application set value that is returned to callback
memory_warn_size[in] ] Memory heap usage (in bytes) that will trigger the callback

Reports on different metrics (memory heap and number of allocations).

Parameters
op[in] This determines which internal metrics to return. Can be either GNSDK_INTERNALS_OP_MEMORY_HEAP or GNSDK_INTERNALS_OP_MEMORY_ALLOCS.
p_current[in] Current value for metric (as reflected by op param)
p_highwater[in] Highest value achieved for metric (as reflected by op param)
