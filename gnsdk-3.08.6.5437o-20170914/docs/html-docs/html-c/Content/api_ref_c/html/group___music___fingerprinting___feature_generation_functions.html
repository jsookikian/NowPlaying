GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_dsp_feature_audio_begin ( gnsdk_user_handle_t  user_handle,
gnsdk_cstr_t  feature_type,
gnsdk_uint32_t  audio_sample_rate,
gnsdk_uint32_t  audio_sample_size,
gnsdk_uint32_t  audio_channels,
gnsdk_dsp_feature_handle_t *  p_feature_handle 
)

Create a DSP Feature handle to begin Feature generation. Applications can create any number of handles to generate Features simultaneously.

Parameters
user_handleThe user handle
feature_typeThe kind of DSP feature, for example a fingerprint.
audio_sample_rateThe source audio sample rate.
audio_sample_sizeThe source audio sample size.
audio_channelsThe source audio channels
p_feature_handleThe handle of the generated DSP feature
GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_dsp_feature_audio_write ( gnsdk_dsp_feature_handle_t  feature_handle,
const gnsdk_byte_t *  audio_data,
gnsdk_size_t  audio_data_bytes,
gnsdk_bool_t *  pb_complete 
)

Write a buffer of audio to the Feature handle to be processed. Audio must be of the format specified in gnsdk_dsp_feature_audio_begin. Audio buffers must be contiguous from the source in order to generate an accurate feature.

Parameters
feature_handleFeature handle instance created by gnsdk_dsp_feature_audio_begin
audio_databuffer of audio data provided by application
audio_data_bytessize of audio buffer in bytes provided by audio_data parameter
pb_completepoint to gnsdk_bool_t to receive 'true' when Feature generation is complete For some Features it is possible that the pb_complete parameter will not be set to true during any amount of writes. This case occurs when the generated Feature needs to utilize all provided audio and the application should provide audio until the source completes.
GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_dsp_feature_end_of_write ( gnsdk_dsp_feature_handle_t  feature_handle)

Indicate to Feature handle that the source audio has reached its end. This will finalize the Feature and prepare it for retrieval. If the complete flag was set to true during the write operations, it is not neccessary to call this function. However calling this API when the write complete flag was set to true is ok and will not have adverse affects.

Parameters
feature_handleFeature handle instance created by gnsdk_dsp_feature_audio_begin

GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_dsp_feature_retrieve_data ( gnsdk_dsp_feature_handle_t  feature_handle,
gnsdk_dsp_feature_qualities_t *  p_feature_qualities,
gnsdk_cstr_t *  p_feature_data 
)

Get the DSP feature qualities and data. Note that if there were no processing errors, but there wasn't enough non-silent audio to generate a fingerprint, the function will return 'success' but the output data param may be set to GNSDK_NULL.

Parameters
feature_handleFeature handle instance created by gnsdk_dsp_feature_audio_begin
p_feature_qualitiesPointer to receive quality flags for Feature
p_feature_dataPointer to receive Feature data in string form. The returned string in p_feature_data is managed by the feature_handle and is released when feature_handle is released.

GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_fp_data_render ( const gnsdk_fp_data_handle_t *  fingerprint_data_handles,
gnsdk_uint32_t  data_handle_count,
gnsdk_uint32_t  render_flags,
gnsdk_str_t *  p_fingerprint_string 
)

Get the fingerprint qualities and renderd fingerprint for the input data handles. Free the returned string with gnsdk_fp_string_free(). For CX fingerprints, if you pass in silent data handles, you will either get an error of FPWARN_SilentAudio returned.

Parameters
fingerprint_data_handles[in] An array of fingerprint data handles.
data_handle_count[in] The count of data handles in the input array.
flags[in] For future use. Pass in 0x0000000 for now.
p_fingerprint_string[out] Pointer to receive the fingerprint string.

GNSDK_EXPORTED gnsdk_error_t GNSDK_API gnsdk_fp_fingerprinter_create ( gnsdk_fp_type_t  fingerprint_type,

Creates the fingerprinter handle. Please provide audio that is 44100Hz stereo with 16bit int or 32bit float samples.

Parameters
fingerprint_type[in] The fingerprint type string (as provided by Gracenote)
audio_sample_rate[in] The source audio sample rate.
audio_format[in] The source audio format (32bit float or 16 bit int).
audio_channels[in] The source audio channels
data_callback_fn[in] Callback function to receive fingerprint data as it is generated.
data_callback_user_data[in] User data to be passed to the callback when it is called.
p_fingerprinter_handle[out] The handle of the fingerprinter
