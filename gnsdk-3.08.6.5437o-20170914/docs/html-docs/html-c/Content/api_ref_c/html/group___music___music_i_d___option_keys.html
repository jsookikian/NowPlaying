#define GNSDK_MUSICID_OPTION_ADDITIONAL_CREDITS

Indicates that additional credits are enabled and will be returned from service.

#define GNSDK_MUSICID_OPTION_ENABLE_AUDIO_SUITABILITY_PROCESSING

This option is experimental and subject to change When enabled audio is processed to determine a suitability rating for music identification. Audio suitability processing is turned off by default. To turn on set this option with value GNSDK_VALUE_TRUE.

#define GNSDK_MUSICID_OPTION_ENABLE_CLASSICAL_DATA

Indicates whether a response should include any associated classical music data.

Remarks:

To retrieve classical music data, define this option key with a True value for a specific MusicID query handle. This retrieves the data and automatically renders it into the XML output. An application's license must be entitled to retrieve this specialized data. Contact your Gracenote Professional Services representative with any questions about this enhanced functionality.

#define GNSDK_MUSICID_OPTION_ENABLE_CONTENT_DATA

Indicates whether a response should include data for use in fetching content (like images).

Remarks:

To allow the GDO to be efficiently used to retrieve content data, define this option key with a True value for a specific MusicID query handle. An application's client ID must be entitled to retrieve this specialized data. Contact your Gracenote Professional Services representative with any questions about this enhanced functionality.

#define GNSDK_MUSICID_OPTION_ENABLE_DSP_DATA

Deprecated, will be removed in a future release. Use GNSDK_MUSICID_OPTION_ENABLE_SONIC_DATA instead.

#define GNSDK_MUSICID_OPTION_ENABLE_EXTERNAL_IDS

Indicates whether a response should include external IDs (third-party IDs).

Remarks:

External IDs are third-party IDs associated with the results (such as an Amazon ID), configured specifically for your application. An application's client ID must be entitled to retrieve this specialized data. Contact your Gracenote Professional Services representative with any questions about this enhanced functionality. External IDs are supported in the following types, for the listed child and value GDOs: GNSDK_GDO_TYPE_ALBUM GNSDK_GDO_CHILD_EXTERNAL_ID GNSDK_GDO_TYPE_TRACK GNSDK_GDO_CHILD_EXTERNAL_ID

#define GNSDK_MUSICID_OPTION_PREFERRED_LANG

Indicates the preferred language of the returned results.

#define GNSDK_MUSICID_OPTION_RESULT_PREFER_COVERART

Indicates use cover art to prefer the returned results.

Remarks:

To set or clear this option use GNSDK_VALUE_TRUE or GNSDK_VALUE_FALSE.

#define GNSDK_MUSICID_OPTION_RESULT_PREFER_XID

Indicates the custom non-Gracenote identifier used to prefer the returned results. Only available where a single result is requested with GNSDK_MUSICID_OPTION_RESULT_SINGLE.

Note:

If you specify both PREFER_XID and PREFER_COVER in an album query, and there is no single match that satisfies both criteria, Gracenote Service returns the PREFER_COVER match first, followed by the PREFER_XID match.If you prefer to use cover art provided by an external partner, consider using only PREFER_XID in the query. Note that there is no guarantee that the album returned through the XID match will contain cover art.

#define GNSDK_MUSICID_OPTION_RESULT_RANGE_SIZE

Indicates whether a response can return up to a specified maximum number of results.

Remarks:

This option is useful for paging through results.

Note:

Gracenote Service limits the range size for some queries. If you specify a range size greater than the limit, the results are constrained. Additionally, neither Gracenote Service nor GNSDK returns an error about this behavior.

Important:

The number of results actually returned for a query may not equal the number of results initially requested. To accurately iterate through results, always check the range start, end, and total values and the responses returned by Gracenote Service for the query (or queries). Ensure that you are incrementing by using the actual last range end value plus one (range_end+1), and not by simply using the initial requested value.

#define GNSDK_MUSICID_OPTION_RESULT_RANGE_START

Indicates whether a response must return a range of results that begin at a specified value.

Remarks:

This option is useful for paging through results.

Note:

Gracenote Service enforces that the range start value must be less than or equal to the total number of results. If you specify a range start value that is greater than the total number of results, no results are returned.

Note:

If you specify both PREFER_XID and PREFER_COVER in an album query, and there is no single match that satisfies both criteria, Gracenote Service returns the PREFER_COVER match first, followed by the PREFER_XID match.If you prefer to use cover art provided by an external partner, consider using only PREFER_XID in the query. Note that there is no guarantee that the album returned through the XID match will contain cover art.

#define GNSDK_MUSICID_OPTION_RESULT_SINGLE

Indicates whether a response must return only the single best result.

Remarks:

The MusicID library selects the single best result based on the query type and input. To set or clear this option use GNSDK_VALUE_TRUE or GNSDK_VALUE_FALSE.

#define GNSDK_MUSICID_OPTION_REVISION_CHECK

Indicates whether a response must return results only when an updated revision exists.

Remarks:

To set or clear this option use GNSDK_VALUE_TRUE or GNSDK_VALUE_FALSE.
